# RNG-ML

Machine-learning based classification of random number generator (RNG) output using statistical features extracted from fixed-size byte windows.

The goal of this project is to investigate whether different classes of random number generators can be distinguished using statistical analysis and supervised machine learning.

---

## Research goal

Random number generators are widely used in simulation, statistics, and cryptography. While many generators aim to produce sequences that appear statistically random, their internal algorithms may still introduce detectable patterns.

This project explores whether machine learning models can identify the generator class based solely on statistical features extracted from the generated byte sequences.

---

## Research questions

1. Can weak pseudorandom number generators be distinguished from stronger generators using machine learning?
2. Which statistical features are most informative for distinguishing generator classes?
3. How do window size and feature selection affect classification performance?

---

## Generator classes

The dataset includes output from three categories of random number generators.

### Weak PRNG baseline

`lcg`  
Linear Congruential Generator implemented in Python.  
Included as a baseline example of a statistically weak generator.

### Modern PRNG

General-purpose pseudorandom generators:

- `mt` – Mersenne Twister  
- `pcg64` – Permuted Congruential Generator

### Cryptographically secure RNG

Generators designed for cryptographic use:

- `urandom` – operating system entropy source
- `secrets` – Python cryptographic random interface

---

## Method overview

The workflow follows a standard machine-learning pipeline for statistical sequence analysis. The analysis pipeline consists of the following steps:

1. Generate byte streams from multiple RNG implementations.
2. Split the streams into fixed-size windows.
3. Extract statistical features from each window.
4. Train machine learning models on the extracted features.
5. Evaluate the models and visualize the results.

---

## Extracted statistical features

The feature set contains statistical diagnostics commonly used in randomness testing and signal analysis. 

| Feature | Description |
|------|------|
| Shannon entropy | Measures information content and distribution uniformity |
| Chi-square statistic | Tests deviation from uniform byte distribution |
| Bit balance | Ratio of 0 and 1 bits |
| Byte autocorrelation | Correlation between consecutive byte values |
| Bit autocorrelation | Correlation between consecutive bits |
| Spectral flatness | Frequency spectrum uniformity |
| Spectral peak ratio | Strength of dominant spectral peak |

These features capture different statistical properties of the sequences, including distribution uniformity, correlation structure, and frequency-domain characteristics.

---

## Machine learning models

Two supervised classification models are currently implemented.

- Logistic Regression
- Random Forest

These models were selected because they provide both interpretable coefficients and feature importance estimates.

## Running the pipeline

Install dependencies:

```bash
pip install -r requirements.txt
```
Generate data and run the analysis:

```bash
python src/generate_streams.py
python src/extract_features.py
python src/train_models.py
python src/plots.py
```

## Output

The pipeline produces several outputs:

- extracted feature dataset  
  `data/features/features.parquet`

- classification reports (precision, recall, F1-score)

- confusion matrices

- feature importance plots

Outputs are saved in `data/results/`.


The results directory is ignored in version control because it contains generated experiment outputs.

## Example outputs

The analysis produces figures such as:

- confusion matrices for each classifier
- feature importance rankings
- statistical feature distributions

These visualizations illustrate how different generators can be distinguished based on statistical characteristics.

---

## Key observations

Preliminary experiments suggest that:

- Weak generators such as **LCG** can be reliably distinguished from stronger generators.
- Several statistical features contribute jointly to classification performance.
- Modern PRNGs and cryptographically secure RNGs are statistically much harder to separate.

---

## Limitations

This project focuses on relatively simple statistical features and fixed window sizes.

Future work could include:

- experimenting with different window sizes
- expanding the feature set
- applying sequence-based models
- comparing additional RNG implementations
